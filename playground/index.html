03 · Guide

A shared rulebook for a diverse playground.

Playground is not only a gallery.

AIGC-X Playground is an open activity hub for competitions, demos, tools, research showcases, and future services.

Open full guide
Types

Many forms

Static designs, interactive experiments, AI demos, docs, tools, prototypes, and future services.

Structure

One entry, one folder

Use playground/submissions/<entry-id>/index.html and keep local assets nearby.

Metadata

Listed by JSON

Add title, author, description, tags, and URL in submissions.json.

Safety

Sandbox first

No secrets, hidden redirects, credential collection, payment forms, or unreviewed trackers.

Backend

Frontend here, API later

Future services can use Playground for UI while backend routes are reviewed separately.

Review

Submit through PR

Entries are reviewed before public deployment so the website can stay open and stable.

04 · Submit

One folder, one HTML page.

Step 01

Create your folder

Put your work in playground/submissions/your-name/index.html.

Step 02

Add metadata

Add title, author, description, tags, and URL in playground/submissions.json.

Step 03

Open a PR

After review and merge, your page appears on the public website automatically.
